# سجل القرارات التقنية (ADR)

سجل مختصر للقرارات المعمارية المعتمدة. كل قرار جديد يُضاف كصف في الجدول، وإذا تم استبداله نُغيّر الحالة إلى "مستبدل" مع إشارة للقرار الجديد.

## كيفية إضافة قرار جديد

1. أضف صفاً جديداً في الجدول بالرقم التالي.
2. اشرح "السبب" بسطر واحد واضح.
3. ضع التاريخ بصيغة `YYYY-MM-DD`.
4. الحالة: `معتمد` / `قيد التجربة` / `مستبدل بـ #N` / `مرفوض`.
5. إن استبدلت قراراً سابقاً، لا تحذفه — غيّر حالته فقط.

## قرارات مرفوضة (للمرجعية)

| القرار | السبب الرفض |
|---|---|
| تخزين الأدوار داخل جدول `profiles` | يفتح ثغرة privilege escalation |
| استدعاء Supabase مباشرة من المكوّنات | يخلط UI بالبيانات ويُصعّب الاختبار |
| تنفيذ embeddings/RAG في الواجهة | يكشف المفاتيح ويُبطئ المتصفح |
