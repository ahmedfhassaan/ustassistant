

# نظام التقييم التفصيلي عند الضغط على "غير مفيد"

## الوضع الحالي
- زر "غير مفيد" يرسل `is_helpful: false` مباشرة إلى جدول `message_feedback` بدون أسباب
- جدول `message_feedback` يحتوي فقط على: `user_id`, `message_content`, `is_helpful`, `created_at`
- لا يوجد ربط بالسؤال الأصلي أو المصادر

## التعديلات المطلوبة

### 1. تعديل قاعدة البيانات — جدول `message_feedback`
إضافة أعمدة جديدة:
- `reason` (text, nullable) — السبب المختار
- `reason_other` (text, nullable) — النص الحر عند اختيار "سبب آخر"
- `notes` (text, nullable) — ملاحظات اختيارية
- `question_content` (text, nullable) — السؤال الأصلي
- `sources` (text, nullable) — المصادر المرفقة بالإجابة

### 2. مكوّن نافذة التقييم — `FeedbackDialog.tsx`
مكوّن Dialog جديد يحتوي على:
- قائمة أسباب (radio): الإجابة غير صحيحة، غير واضحة، ناقصة، لا يوجد مصدر، سبب آخر
- حقل نص يظهر فقط عند اختيار "سبب آخر"
- حقل ملاحظات اختياري
- زرا إرسال وإلغاء
- يستخدم مكوّنات `Dialog` و `RadioGroup` و `Textarea` الموجودة

### 3. تعديل `ChatMessage.tsx`
- عند الضغط على "غير مفيد": فتح `FeedbackDialog` بدلاً من الإرسال المباشر
- تمرير بيانات الرسالة (المحتوى، المصدر) والسؤال السابق للمكوّن
- بعد الإرسال: عرض حالة "تم التقييم" وتعطيل الأزرار
- زر "مفيد" يبقى كما هو (إرسال مباشر)

### 4. تمرير السؤال — تعديل `Message` interface
إضافة `question?: string` اختياري إلى واجهة `Message` لربط كل رد بالسؤال الذي سبقه، يتم تعبئته في `Chat.tsx` عند بناء رسالة المساعد.

### التفاصيل التقنية
- منع التكرار: `feedback !== null` يمنع إعادة التقييم (موجود حالياً)
- الأسباب مخزنة كنص عربي في عمود `reason`
- بيانات التقييم الكاملة: `user_id`, `message_content`, `is_helpful`, `reason`, `reason_other`, `notes`, `question_content`, `sources`
- هذا الهيكل يدعم لاحقاً بناء لوحة تحليل للأسباب والمصادر الأكثر إشكالية
