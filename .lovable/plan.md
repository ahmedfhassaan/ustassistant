# منع تشويش إجابات القبول والمقررات بمشاريع التخرج

## المشكلة
عند سؤال الطالب عن **القبول والتسجيل** أو **المقررات والخطط الدراسية**، قد يستخرج RAG محتوى من ملفات مشاريع التخرج (مثل "نظام إدارة المحتوى الطلابي - زميل"، "مشروع متجر للتجارة الإلكترونية"، إلخ) لأن بعضها يذكر كلمات شبيهة (تسجيل، مقرر، نظام، طلاب). هذا يُسبب إجابات غير دقيقة أو مشوّشة.

## الحل
إضافة **مُصنِّف نية** بسيط في `supabase/functions/chat/index.ts` يحدد إذا كان السؤال متعلقاً بـ:
- **القبول والتسجيل** (تسجيل، قبول، تقديم، التحاق، شروط القبول، رسوم القبول…)
- **المقررات والخطط الدراسية** (خطة دراسية، مقرر، مادة، ساعات معتمدة، جدول، تخصص، رمز مقرر…)

عند مطابقة النية، يتم **استبعاد ملفات مشاريع التخرج** من نتائج RAG قبل بناء سياق الإجابة.

## كيف نتعرف على ملفات مشاريع التخرج
عند الفحص، أسماء ملفاتها تبدأ عادةً بأحد: `مشروع`, `نظام`, `تطبيق`, `منصة`, `موقع`, `بوابة`. بينما الملفات الرسمية تبدأ بـ: `دليل طالب`, `الخطة الدراسية`, `الخطة الدراسة`, `لائحة`، إلخ.

سيتم تحديد المشروع بقاعدة بسيطة:
```
isGraduationProject = اسم الملف يبدأ بأحد كلمات (مشروع/نظام/تطبيق/منصة/موقع/بوابة)
                      AND لا يحتوي على كلمات (دليل/خطة/لائحة/سياسة/رسوم)
```

## التغييرات في الكود

**ملف واحد فقط:** `supabase/functions/chat/index.ts`

1. إضافة دالة `isAdmissionOrCurriculumQuestion(text)` بجانب `classifyQuestion`.
2. إضافة دالة `isGraduationProjectDoc(name)` لكشف ملفات المشاريع.
3. عند بناء `chunks` (السطر ~513)، إذا كان السؤال من فئة القبول/المقررات، نُطبّق فلتراً جديداً يستبعد chunks المشاريع — مع ضمان عدم استبعاد كل النتائج (إن أصبحت فارغة بعد الفلتر، نُسجل تحذيراً ونتركها لينطلق Live Search تلقائياً).
4. إضافة سجل تشخيصي:
```
[chat] intent=admission_or_curriculum: filtered N project chunks, kept M
```

## لا تغييرات في
- قاعدة البيانات (لا migrations)
- واجهة المستخدم
- إعدادات الـ admin

## النتيجة المتوقعة
- سؤال "ما هي شروط القبول؟" → يأخذ فقط من "دليل الطالب" و"اللوائح".
- سؤال "ما هي مقررات تقنية المعلومات؟" → يأخذ فقط من "الخطة الدراسية".
- سؤال "اذكر مشاريع التخرج السابقة" → يأخذ من ملفات المشاريع كالمعتاد (الفلتر لا ينطبق).
- سؤال "هل يوجد نظام ERP" → الفلتر لا ينطبق، Live Search يعمل كالمعتاد.
