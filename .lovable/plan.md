

# تحسين رسائل الخطأ والحالات الفارغة

## الوضع الحالي
معظم الصفحات تعرض نصاً بسيطاً مثل "لا توجد أسئلة بعد" أو "لم يتم رفع أي مستندات" بدون أي عناصر بصرية توضيحية — مما يجعل التجربة ضعيفة عند عدم وجود بيانات.

## التحسينات المطلوبة

### 1. إنشاء مكوّن `EmptyState` موحّد
مكوّن قابل لإعادة الاستخدام يعرض:
- أيقونة كبيرة توضيحية (مثلاً مستند فارغ، علامة استفهام)
- عنوان رئيسي واضح
- نص وصفي يشرح السبب أو يقترح إجراء
- زر إجراء اختياري (مثل "رفع مستند" أو "ابدأ محادثة")
- دعم الوضع الفاتح والداكن

### 2. تطبيق المكوّن على الصفحات التالية

| الصفحة | الحالة الفارغة | الأيقونة | الإجراء المقترح |
|--------|---------------|---------|----------------|
| لوحة التحكم (أكثر الأسئلة شيوعاً) | لا توجد أسئلة | MessageSquare | — |
| قاعدة المعرفة | لا توجد مستندات | FileText | زر "رفع مستند" |
| قاعدة المعرفة | لا توجد نتائج بحث | Search | زر "مسح البحث" |
| الأسئلة الشائعة | لا توجد أسئلة مسجلة | HelpCircle | — |
| الدردشة (شاشة الترحيب) | — | موجودة بالفعل ✓ | — |

### 3. تحسين رسائل خطأ تسجيل الدخول
- إضافة أيقونة تحذير بجانب رسالة الخطأ في `Login.tsx`
- تحسين تصميم رسالة الخطأ بإطار ملوّن وخلفية خفيفة

### 4. إضافة حالة خطأ عند فشل تحميل البيانات
- عرض رسالة خطأ مع زر "إعادة المحاولة" في Dashboard و FAQ عند فشل الاتصال

## التفاصيل التقنية
- إنشاء `src/components/EmptyState.tsx` بخصائص: `icon`, `title`, `description`, `actionLabel?`, `onAction?`
- تحديث `AdminDashboard.tsx`, `AdminKnowledge.tsx`, `AdminFAQ.tsx` لاستخدام المكوّن الجديد
- تحديث `Login.tsx` لتحسين عرض رسالة الخطأ
- إضافة معالجة `isError` من `useQuery` في Dashboard و FAQ
