
## تطبيق التحسينات على صفحة التوثيق

تعديل **`src/pages/Documentation.tsx` فقط** (لا تغييرات على Edge Functions، DB، أو design tokens). الهدف: استبدال العبارات الغامضة بأرقام قابلة للقياس، تصحيح التناقضات، وإضافة معلومات ينقصها المطوّر.

---

### 1) قسم RAG (`#rag`) — إعادة هيكلة جدول المراحل

- **كاش هاش**: إضافة "TTL افتراضي ٢٤ ساعة، نطاق آمن `1h–72h`"، وذكر تخزين `source_set_hash` لتجنّب الإجابة من كاش بمصادر قديمة.
- **كاش دلالي**: تثبيت العتبة الافتراضية `≥ 0.92` cosine similarity، مع شارة "إجابة مخزَّنة" في الواجهة (ملاحظة فقط، لا تنفيذ UI الآن).
- **إعادة الصياغة**: مهلة `3s` صارمة، fallback للسؤال الأصلي عند الفشل.
- **البحث الهجين**: إيضاح أن `0.4 FTS / 0.6 Semantic` تُجمع بعد min-max normalization على نفس السلَّم `[0,1]`، مع fallback لـ FTS وحده عند غياب الـ embedding.
- **عتبة الثقة**: قيمة افتراضية `0.62`، نطاق `[0.5, 0.8]`، أقل من ذلك = اعتذار صريح.
- **التوليد**: مهلة `25s` على Gemini، ميزانية إجمالية `30s` للطلب.

استبدال البطاقات الثلاث الصغيرة بأربع:
| البطاقة | القيمة |
|---|---|
| حجم المقطع | `600 token` مع تداخل `80 token` |
| أبعاد المتجه | `768` عبر `outputDimensionality` (الافتراضي ٣٠٧٢) |
| ميزانية الطلب | `p50 ≤ 1.2s` (كاش) / `p95 ≤ 6s` (RAG كامل) |
| منع الهلوسة | غياب مصادر بثقة `< 0.62` ⇒ اعتذار |

### 2) قسم تسجيل الدخول والصلاحيات (`#auth`) — تصحيح التناقض

- توضيح أن `20260000` هو **Seed** أوّلي يُنشأ في `user_roles` بالدور `admin`، وليس فحصاً مكتوباً في الكود.
- إضافة سطر: "ترقية أي مستخدم لاحقاً تتم عبر migration على `user_roles`، لا عبر الواجهة".
- استبدال جملة "بدون إمكانية إنشاء مشرفين إضافيين من الواجهة" (في قسم الأمان) بـ: "إنشاء/إلغاء صلاحية المشرف يتم عبر migration فقط".

### 3) قسم الأمان (`#security`) — حذف ادعاءات غير دقيقة وإضافة ما ينقص

- **حذف**: bullet "تطهير المدخلات (trim) لمنع الثغرات" — `trim` ليس إجراءً أمنياً.
- **إضافة Rate Limiting**: حد افتراضي `30 طلب/دقيقة لكل حساب` على دالة `chat` (يُنفَّذ لاحقاً في Edge، التوثيق فقط الآن).
- **توضيح bcrypt**: cost factor `10`، تتم المقارنة داخل Edge Function `login`، لا تُرسَل التجزئة للواجهة.
- **إضافة**: "المتجهات المُولَّدة تُرفض إن لم تكن بطول ٧٦٨ بُعداً" كفحص دفاعي.

### 4) قسم الأداء (`#performance`) — جدول SLO قابل للقياس

استبدال الفقرة العامة بجدول جديد:

| المقياس | الهدف |
|---|---|
| زمن إجابة من الكاش (p50) | `≤ 1.2s` |
| زمن إجابة RAG كامل (p95) | `≤ 6s` |
| ميزانية طلب قصوى | `30s` (تنتهي بإجابة جزئية أو اعتذار) |
| مهلة embedding | `4s` |
| مهلة rewrite-query | `3s` |
| مهلة Gemini generation | `25s` |
| دقة الإجابات على `golden_eval` | `≥ 85%` (تشابه دلالي ≥ 0.85) |

### 5) قسم حدود النظام (`#limits`) — إضافات

- إضافة: "نموذج `gemini-3-flash-preview` نموذج معاينة؛ ستتم ترقيته عند إصدار GA دون كسر العقد العام للـ Edge Function".
- إضافة: "مشاريع التخرج تُستبعد تلقائياً من نوايا: قبول، تسجيل، مقررات، رسوم".

### 6) قسم ملاحظات المطورين (`#developers`) — إضافة عقد دالة `chat`

إضافة جدول جديد بعقد الإدخال/الإخراج:

| الحقل | النوع | ملاحظة |
|---|---|---|
| `question` (in) | string | يُمرَّر بعد `normalizeArabic` |
| `user_id` (in) | uuid | إجباري لتتبع الاستخدام |
| `conversation_id` (in) | uuid? | اختياري لمتابعة المحادثة |
| `cache_hit` (out) | `'hash' \| 'semantic' \| 'none'` | شفافية الكاش |
| `confidence` (out) | number | `[0,1]` |
| `sources` (out) | `Array<{type, title, ref}>` | قد تكون فارغة |
| البث | SSE | أحداث `delta` ثم `done` |
